# T3522 — Deregistration Accept-Wait and Retransmission

> Built 2026-07-03 on branch `feat/t3522`. Removes the "immediate cleanup"
> deviation from [38](38-network-dereg.md): the network-initiated Deregistration
> Request is now a real supervised procedure (TS 24.501 §5.5.2.3.4).

## What was built

- **`DeregCmd`** — the association channel carries `Start(ue)` (from the SBI
  withdrawal callback) or `T3522Expiry(ue)` (from the timer); `serve_gnb`'s
  `select!` dispatches both.
- **`on_network_deregistration`** (reworked) — sends the Deregistration Request
  (UE terminated), marks `UeContext.dereg_attempts = 1`, arms **T3522**
  (`T3522_SECS = 6`), and **keeps the contexts**. The directory entry drops
  immediately (no duplicate withdrawals); a duplicate `Start` is ignored while
  the procedure runs. An unsecured UE still gets the immediate-release fallback.
- **`on_t3522_expiry`** — while under `T3522_MAX_SENDS` (5 = initial + 4
  retransmissions) re-protect + retransmit and re-arm; on exhaustion, abort:
  UEContextReleaseCommand and the contexts are dropped anyway. Stale expiries
  (procedure completed) are no-ops — no timer cancellation needed.
- **Deregistration Accept (UE terminated)** — new `dispatch_uplink_nas` arm:
  an accept while the procedure runs completes it (release command + context
  drop); an unexpected accept is logged and ignored. `nas::
  deregistration_accept_to_ue()` builder for the UE side/tests.
- Timer duration is a parameter of both flow functions (production passes
  `T3522_SECS`) — what makes the machinery testable without waiting.

## Verification

- `cargo test --workspace --exclude bdd` — green. Reworked/new:
  - `subscription_withdrawal_deregisters_the_ue` — Start sends only the request
    (context kept, `attempts = 1`, duplicate Start ignored); the UE's accept
    completes with the release command and drops the context; a stale expiry
    after completion is a no-op.
  - `t3522_retransmits_then_aborts` — instant timer proves the expiry lands on
    the channel; expiries 2..=5 retransmit with the attempt count advancing;
    the sixth aborts with the release command and drops the context.
- **Live loopback demo** (free-ran-ue never answers a UE-terminated dereg):
  `curl DELETE` → AMF log shows attempt 1, retransmissions at exact 6-second
  intervals (attempts 2–5), then "T3522 exhausted after 5 transmissions —
  aborting" and the UEContextReleaseCommand at t+30s.
- **BDD, 5 scenarios / 25 steps green** (regression).

## Known limitations / next steps

- **T3396/T3346 remain unenforced core-side** (design/30/36 posture) — T3522 is
  the first timer the core itself runs.
- **The PDU session is released at Start** — a UE that answers the first request
  quickly loses nothing; one that never answers has no session for up to 30s of
  limbo either way. Matches the spec's intent (resources go with the dereg).
- UECM serving-AMF tracking, multiple PDU sessions, UE-AMBR, and AMF-side SMF
  selection remain open.
