# Netns datapath BDD test — the forwarded packet, automated

> Built 2026-06-30 on branch `feat/bdd-datapath`. A cucumber/netns integration test that
> reproduces the end-to-end user-plane ping automatically, so the datapath doesn't regress.

The live ping (design/21 follow-up) proved the UPF forwards a real packet, but it was a manual
run with the external free-ran-ue simulator. This slice makes it a **self-contained,
repeatable BDD test** — no external simulator, driven entirely by radiant's own codecs.

## The `bdd/` crate

A workspace member (excluded from the quick `cargo test --workspace --exclude bdd` sweep — it's
privileged and Linux-only). Mirrors the zebra-rs BDD conventions: `@serial`, a feature tag for
resource scoping, `Given a clean test environment`, and a `Scenario: Teardown topology` ending
in `the test environment should be clean`.

- `src/netns.rs` — `ip netns`/veth helpers (create/delete namespace, wire a host↔ns veth,
  spawn a process in the namespace, kill everything in a namespace, list-by-prefix for sweeps).
- `src/datapath.rs` — the **SMF** and **gNB** roles: drive PFCP (`pfcp` crate) to establish +
  modify a session, then GTP-U (`gtpu` crate) a crafted ICMP echo through N3 and read the reply.
  Hand-built IPv4/ICMP with correct checksums (unit-tested).
- `tests/cucumber.rs` — the World + steps + runner; `tests/features/n6_datapath.feature`.

## How the test moves a packet (no simulator)

```
 ┌──────────────────────────┐   veth   ┌───────────────────────────────┐
 │ test process (host)      │ 10.0.1.1 │ namespace <tag>_upf            │
 │  SMF: PFCP  → N4          ├──────────┤ 10.0.1.2  nf-upf (real binary) │
 │  gNB: GTP-U ↔ N3          │          │  N4 :8805  N3 :2152            │
 └──────────────────────────┘          │  N6 TUN n6upf0 10.45.0.1/16    │
                                        └───────────────────────────────┘
```

1. Start the real `nf-upf` in the namespace (so its N6 TUN opens — needs `CAP_NET_ADMIN`).
2. Play the SMF: PFCP Association → Session Establishment (UPF allocates the uplink N3 F-TEID
   + records UE IP 10.45.0.2) → Session Modification (install a downlink gNB F-TEID pointing at
   the host veth).
3. Play the gNB: GTP-U-encap an ICMP echo (10.45.0.2 → 10.45.0.1) on the uplink TEID → N3.
4. The UPF decaps → `n6::uplink` (anti-spoof) → writes to `n6upf0`; the **namespace kernel**
   answers the ping; the UPF `n6::downlink`-routes the reply by UE IP → GTP-U → the gNB F-TEID.
5. Receiving the reply proves the full **N3 → N6 → N3** round trip.

The namespace matters: the N6 TUN and the gNB's N3 endpoint get distinct IPs (both use GTP-U
port 2152), and the ping reply is generated by the *namespace's* kernel routing over `n6upf0`.

## Verification

- `cargo test -p bdd --test cucumber` — **2 scenarios / 10 steps pass**, ending in "Test
  environment is clean" (namespaces, veth, TUN, and the UPF process all torn down).
- `cargo test -p bdd --lib` — the ICMP checksum + echo-reply-recognition units pass.
- `cargo test --workspace --exclude bdd` — 49 pass (unchanged); `cargo clippy` clean.

## Known limitations / next steps

- **Privileged + Linux-only** — needs `sudo`/`CAP_NET_ADMIN` and `ip netns`, so it runs on a
  dev host or a privileged CI runner, not the default unit-test sweep.
- **Self-contained, not the full RAN** — the test plays SMF+gNB with radiant's own codecs
  rather than the free-ran-ue simulator; it exercises the UPF datapath (N3/N6, PFCP, GTP-U,
  the real TUN + kernel), not the NGAP/NAS control plane. A simulator-driven BDD feature (with
  free-ran-ue) would cover registration → PDU session → ping in one flow.
- **One session / IPv4 only**, fixed addresses (fine under `@serial`).
